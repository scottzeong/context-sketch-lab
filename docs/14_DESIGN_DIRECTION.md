# v1 디자인 방향

## 1. 디자인 목표

Context Sketch Lab은 일반적인 관리자 CRUD 화면이 아니라 튜터가 수업을 설계하고 검토하는 전문 작업 환경이다.

목표 톤:

```text
교육 연구실
튜터 워크벤치
차분한 전문성
반복 작업에 강한 고밀도 화면
학생에게는 따뜻하고 명확한 화면
보호자에게는 신뢰감 있는 리포트 화면
```

## 2. 정보 구조

튜터 화면은 다음 작업 흐름을 중심으로 구성한다.

```text
글 만들기
글 분석하기
세션 설계하기
학생 제출 보기
관찰 입력하기
AI 초안 검토하기
승인/공개하기
포트폴리오 확인하기
```

대시보드는 장식적인 랜딩 페이지가 아니라 오늘 처리해야 할 실제 작업을 보여준다.

## 3. 핵심 UI 패턴

```text
- 좌측 사이드바: 주요 영역 이동
- 상단 작업 바: 현재 화면의 핵심 액션
- 2열 워크벤치: 왼쪽 원본/입력, 오른쪽 AI 초안/결과
- 상태 배지: draft, generated, reviewed, approved, published
- 편집 가능한 AI 결과 패널
- 튜터 승인 버튼은 항상 명확한 마지막 단계에 배치
- 리포트 화면은 인쇄/공유 가능한 문서형 레이아웃
```

## 4. 화면별 디자인 기준

### Tutor Dashboard

```text
- 오늘 검토할 제출물
- 최근 생성한 글
- 진행 중 세션
- 승인 대기 피드백
- 빠른 생성 버튼
```

### Text Studio

```text
- 조건 입력 폼
- 생성된 글 편집기
- 글 구조 분석 패널
- 세션 만들기 연결
```

### Review Workspace

```text
- 학생 제출 이미지
- 학생 자기 설명
- 튜터 관찰 입력
- AI 평가 초안
- AI 피드백 초안
- 승인/공개 흐름
```

### Portfolio/Report

```text
- 학생별 성장 흐름
- 루브릭 변화
- 튜터 코멘트
- 대표 활동
- 다음 학습 제안
```

## 5. 시각 스타일

```text
- 배경은 밝고 절제된 중립색
- 강조색은 한두 개만 사용
- 카드 남발 금지
- 실제 작업 밀도를 확보
- 긴 문서는 읽기 좋은 폭과 행간 유지
- 상태, 권한, 승인 흐름은 색상과 텍스트를 함께 사용
```

## 6. 품질 기준

```text
- 튜터가 30초 안에 다음 작업을 알 수 있는가?
- AI 결과를 복사하지 않고 바로 수정할 수 있는가?
- 학생/보호자에게 공개되는 문장은 승인 상태가 명확한가?
- 모바일에서도 제출과 피드백 확인이 가능한가?
- 데모가 아니라 실제 운영 화면처럼 보이는가?
```
