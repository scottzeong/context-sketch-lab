# Codex 개발 워크플로우

## 1. 기본 원칙

Codex는 전체 시스템을 한 번에 임의로 확장하지 않고, v1 범위 안에서 기능 단위로 완성한다.

```text
1. 문서를 먼저 기준으로 삼는다.
2. 구현 전 현재 코드와 스키마를 확인한다.
3. 튜터 워크플로우를 우선한다.
4. AI 응답은 Zod schema로 검증한다.
5. OpenAI API key는 서버에서만 사용한다.
6. Supabase service role key는 client에 노출하지 않는다.
7. 이미지 AI 분석 기능은 추가하지 않는다.
```

## 2. 개발 순서

```text
1. 문서 개정
2. 텍스트 AI 스파이크
3. 디자인 시스템과 앱 shell
4. Supabase/Auth
5. 튜터 글 생성/분석
6. 세션 생성
7. 학생 제출
8. 튜터 리뷰
9. 평가/피드백
10. 포트폴리오/리포트
11. 매뉴얼/배포
```

## 3. 코드 변경 체크리스트

```text
- 요청 범위에 없는 기능을 만들지 않았는가?
- AI 이미지 분석이 다시 들어가지 않았는가?
- AI 응답이 schema로 검증되는가?
- 튜터 승인 전 학생에게 최종 결과가 공개되지 않는가?
- 권한 체크가 서버에서 이뤄지는가?
- UI가 실제 튜터 작업에 충분히 빠르고 명확한가?
- 문서와 README가 변경 사항을 반영하는가?
```

## 4. 디자인 검토 체크리스트

```text
- 첫 화면이 실제 작업 화면인가?
- 대시보드가 과도한 마케팅 페이지처럼 보이지 않는가?
- 튜터가 생성, 분석, 리뷰, 승인 흐름을 빠르게 오갈 수 있는가?
- 버튼, 입력, 상태 배지가 일관적인가?
- 결과 JSON이나 AI 초안이 편집 가능한 작업물로 이어지는가?
- 모바일에서 텍스트와 버튼이 겹치지 않는가?
```
