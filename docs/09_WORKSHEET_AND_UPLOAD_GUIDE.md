# 활동지 및 제출 가이드

## 1. 활동지 원칙

활동지는 디지털 보드가 아니라 종이 기반 사고 도구다.

학생은 글을 읽고 종이에 비언어적 맥락스케치를 작성한다. 가능한 한 글보다 그림, 기호, 선, 화살표, 위치, 크기, 거리, 방향으로 표현한다.

## 2. 템플릿 유형

```text
기본형: 직접 정보, 추론 정보, 중심 스케치, 핵심 연결, 오늘의 한 문장
원인-결과형: 사건, 원인, 감정 변화, 결과, 다른 가능성
비교-대조형: A 관점, B 관점, 공통점, 차이점, 새롭게 보인 점
문제-해결형: 겉으로 보이는 문제, 숨은 원인, 영향을 받는 사람, 해결 방향
주장-근거형: 주장, 근거, 반대 가능성, 내 판단, 질문
```

## 3. 학생 제출

학생 제출 항목:

```text
- 스케치 이미지 1장 이상
- 학생 자기 설명
- 가장 중요하다고 생각한 연결
- 어려웠던 부분
```

AI는 이미지를 분석하지 않는다. 이미지는 튜터가 검토하기 위한 원본 자료다.

## 4. 튜터 리뷰 입력 항목

튜터는 제출물을 보고 다음을 입력한다.

```text
- 관찰 메모
- 학생이 잘 표현한 부분
- 핵심 연결
- 오해 또는 빠진 부분
- 다음 활동 제안
- 공개 가능한 피드백 방향
```

이 입력이 AI 평가/피드백 초안의 근거가 된다.

## 5. 업로드 원칙

```text
- 이미지는 Supabase Storage private bucket에 저장한다.
- 이미지 원본 URL을 client DB 필드에 직접 노출하지 않는다.
- submission_images.storage_path를 기준으로 signed URL을 발급한다.
- 튜터와 권한 있는 관리자만 원본 이미지를 볼 수 있다.
```
