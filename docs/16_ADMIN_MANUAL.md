# 관리자 매뉴얼

## 목적

관리자는 Roter Faden의 사용자 계정, 역할, 보호자-학생 연결, 계정 상태, 글 작성 드롭다운 메뉴를 관리합니다.

## 접속

- URL: `/admin/users`
- 필요 role: `admin`
- 관리자 로그인 후 기본 이동 위치도 `/admin/users`입니다.

## 새 계정 생성

1. `새 계정 생성` 영역에서 이메일, 표시 이름, 역할, 임시 비밀번호를 입력합니다.
2. 필요하면 학습 연령대를 선택합니다.
3. `계정 생성`을 누릅니다.
4. 사용자에게 이메일과 임시 비밀번호를 전달합니다.
5. 사용자는 첫 로그인 후 `/onboarding`에서 표시 이름, 학습 관련 정보, 비밀번호를 확인합니다.

## 사용자 관리

관리자는 다음 항목을 수정할 수 있습니다.

- 표시 이름
- role
- 계정 상태
- 학습 연령대

읽기 수준 입력 항목은 운영 화면에서 사용하지 않습니다.

## 비밀번호 재설정

1. 사용자 목록에서 대상 사용자를 선택합니다.
2. `비밀번호 재설정 이메일`을 누릅니다.
3. 사용자는 이메일 링크를 통해 새 비밀번호를 설정합니다.

## 보호자-학생 연결

1. `부모-학생 연결 관리`에서 보호자와 학생을 선택합니다.
2. 관계를 입력합니다.
3. 상태를 `승인`으로 저장합니다.
4. 보호자는 `/parent/dashboard`에서 연결된 학생의 공개 피드백을 볼 수 있습니다.

## 글 작성 드롭다운 관리

관리자는 튜터 워크벤치에서 사용하는 드롭다운 메뉴를 관리합니다.

- 연령
- 난이도
- 분량
- 글구조

운영 방법:

1. 수정할 항목 탭을 선택합니다.
2. 표시 이름, 실제 값, 정렬 순서를 입력해 메뉴를 추가합니다.
3. 기존 메뉴의 표시 이름, 값, 순서, 사용 여부를 수정할 수 있습니다.
4. 더 이상 쓰지 않는 메뉴는 삭제하거나 사용 체크를 해제합니다.

주의:

- 이미 저장된 글의 기존 값은 자동 변경되지 않습니다.
- 수업 중인 튜터가 있다면 메뉴 삭제보다 비활성화를 우선합니다.
- 이 기능은 `supabase/migrations/011_config_options.sql` 적용 후 정상 저장됩니다.

## 운영 주의

- 관리자 계정을 비활성화하기 전에 다른 활성 관리자 계정이 있는지 확인합니다.
- 학생 제출물과 피드백 기록이 있는 계정은 실제 삭제보다 비활성화를 우선합니다.
- 운영 배포 후 `/manual`에서 HTML 매뉴얼 화면도 함께 확인합니다.
