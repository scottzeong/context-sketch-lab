# Roter Faden 사용자 매뉴얼 및 릴리즈 노트

## 제품 개요

Roter Faden은 튜터가 읽기 자료를 생성하고, 학생이 맥락 스케치를 제출하며, 튜터가 피드백과 리포트를 관리하는 학습 운영 도구입니다.

현재 v1의 AI 역할은 다음으로 제한합니다.

- 튜터가 정한 주제와 조건에 맞는 AI 글 생성
- 튜터 직접 글 입력 및 구조 분석
- 생성/저장된 글의 구조 분석
- 튜터 관찰 내용을 바탕으로 피드백/평가 초안 생성

AI는 학생 그림을 최종 판정하지 않습니다. 학생 스케치 해석과 최종 피드백 결정은 튜터가 담당합니다.

## 역할별 사용 흐름

### 관리자

1. `/admin/users`에서 튜터, 학생, 보호자 계정을 생성합니다.
2. 사용자 role, 표시 이름, 연령대, 읽기 수준, 계정 상태를 관리합니다.
3. 보호자와 학생을 연결하고 승인 상태로 변경합니다.
4. 비밀번호 재설정 이메일을 발송하거나 계정을 비활성화합니다.
5. Supabase SQL migration 적용 여부와 Vercel 환경 변수를 점검합니다.

### 튜터

1. `/tutor/workbench`에서 AI로 글을 생성하고 구조를 분석합니다.
2. 결과가 적절하면 Text 저장소에 저장합니다.
3. `/tutor/texts`에서 저장된 글을 검색하고 확인합니다.
4. `/tutor/groups`에서 학습 그룹을 만들고 학생을 배정합니다.
5. `/tutor/sessions/new`에서 글, 그룹, 활동지 유형을 선택해 세션을 생성합니다.
6. `/tutor/sessions`에서 세션을 검색, 공개, 종료, 복제합니다.
7. `/tutor/submissions`에서 학생 제출물을 검토하고 튜터 관찰을 입력합니다.
8. AI 피드백/평가 초안을 생성한 뒤 튜터가 수정하고 공개합니다.
9. `/tutor/reports`에서 학생별 리포트 초안을 편집, 저장, 인쇄/PDF 출력합니다.

### 학생

1. `/student/dashboard`에서 공개된 세션을 확인합니다.
2. 세션 글을 읽고 종이에 맥락 스케치를 작성합니다.
3. 제출 화면에서 스케치 이미지를 업로드하고 자기 설명을 입력합니다.
4. 튜터가 피드백을 공개하면 피드백 상세 화면에서 확인합니다.
5. `/student/portfolio`에서 공개된 피드백과 성장 기록을 누적 확인합니다.

### 보호자

1. 관리자에게 학생과의 연결 승인을 요청합니다.
2. `/parent/dashboard`에서 연결된 학생의 공개 피드백을 확인합니다.
3. 학생별 필터로 최근 피드백과 누적 기록을 봅니다.
4. 보호자 요약 리포트를 인쇄하거나 PDF로 저장합니다.

## 릴리즈 노트 v1

### 핵심 기능

- Supabase Auth 기반 role 라우팅
- 관리자 글 작성 드롭다운 설정
- HTML 매뉴얼 화면
- 관리자 계정 생성 및 온보딩
- 부모-학생 연결 관리
- 튜터 AI Workbench
- Text 저장소
- 학습 그룹 관리
- 세션 생성, 검색, 복제, 공개/종료
- 학생 제출 및 Supabase Storage 이미지 업로드
- 튜터 리뷰 워크스페이스
- AI 피드백/평가 초안
- 학생 피드백 확인 및 포트폴리오
- 보호자 대시보드
- 학생 리포트 편집, 저장, 인쇄/PDF

### 운영상 주의

- `SUPABASE_SERVICE_ROLE_KEY`는 서버 환경 변수에만 저장해야 합니다.
- Supabase SQL migration은 번호 순서대로 적용해야 합니다.
- `submission-images` bucket은 private bucket입니다.
- 피드백 공개 전에는 학생/보호자에게 피드백이 보이지 않아야 합니다.
- AI 출력은 튜터 검토용 초안이며 최종 판단으로 사용하지 않습니다.
