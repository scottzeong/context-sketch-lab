# Roter Faden 사용자 매뉴얼 및 릴리즈 노트

## 제품 개요

Roter Faden은 튜터가 수업용 글을 작성하고, 학생이 맥락 스케치를 제출하며, 튜터가 피드백과 리포트를 관리하는 학습 운영 도구입니다.

v1에서 AI는 학생 그림을 직접 판정하지 않습니다. 그림과 맥락 해석은 튜터가 담당하고, AI는 다음 작업을 돕습니다.

- 튜터가 정한 조건에 맞는 글 생성
- 튜터가 직접 입력한 글의 구조 분석
- 저장된 글의 구조 분석
- 튜터 관찰 기반 평가와 피드백 초안 생성
- 누적 공개 피드백 기반 리포트 초안 작성 보조

## 역할별 사용 흐름

### 관리자

1. `/admin/users`에서 사용자 계정과 역할을 관리합니다.
2. Settings에서 운영 옵션을 관리합니다.
3. 연령, 난이도, 분량, 글구조, 루브릭 평가구조, 루브릭 가중치를 관리합니다.
4. 글구조와 루브릭 평가구조에는 AI 프롬프트에 사용할 설명을 직접 입력합니다.
5. Texts, Classes, Sessions, Reports를 참조하여 운영 현황을 확인합니다.

관리자는 새 세션 생성, AI Workbench, 제출물 리뷰를 직접 수행하지 않습니다.

### 튜터

1. `/tutor/workbench`의 `TEXT ARCHITECTURE`에서 `AI Generation` 또는 `Text Writing`을 선택합니다.
2. 생성 또는 입력한 글을 구조 분석하고 저장합니다.
3. `/tutor/texts`에서 저장된 글과 구조 분석을 확인합니다.
4. `/tutor/groups`에서 학습 Class를 만들고 학생을 배정합니다.
5. `/tutor/sessions/new`에서 글, Class, 활동지 유형을 선택해 세션을 생성합니다.
6. `/tutor/sessions`에서 세션 상태를 `초안`, `공개`, `종료`로 관리합니다.
7. `/tutor/submissions`에서 학생 제출물과 이미지를 확인하고 튜터 관찰을 입력합니다.
8. AI 피드백/평가 초안을 만든 뒤 튜터가 수정하여 최종본만 공개합니다.
9. `/tutor/reports`에서 학생 리포트를 편집, 저장, 복사, 인쇄/PDF 출력합니다.

### 학생

1. `/student/dashboard`에서 공개된 세션을 확인합니다.
2. 글을 읽고 종이나 화면에 맥락 스케치를 작성합니다.
3. 제출 화면에서 이미지와 자기 설명을 업로드합니다.
4. 튜터가 피드백을 공개하면 피드백 상세 화면에서 확인합니다.
5. `/student/portfolio`에서 누적 성장 기록을 확인합니다.

### 보호자

1. 관리자에게 학생 연결 승인을 요청합니다.
2. `/parent/dashboard`에서 연결된 학생의 공개 피드백을 확인합니다.
3. 보호자 요약 리포트를 인쇄하거나 PDF로 저장합니다.

## 릴리즈 노트 v1

### 포함 기능

- 브랜드 프론트 페이지 `/`
- HTML 매뉴얼 페이지 `/manual`
- Supabase Auth 기반 role 라우팅
- 관리자 계정 생성과 권한 관리
- 관리자 운영 옵션 관리
- 글구조/루브릭 평가구조 프롬프트 입력
- 루브릭 가중치 관리
- 튜터 AI Workbench
- AI 글 생성 및 튜터 직접 글 입력
- Text 저장소와 구조 분석 보기
- 학습 Class 관리
- 세션 생성, 공개, 종료, 복제
- 학생 제출 UX 및 이미지 업로드
- 튜터 리뷰 Workspace
- 학생 피드백 상세 화면
- 학생 포트폴리오
- 튜터 리포트 편집/저장/인쇄/PDF
- 보호자 대시보드 및 요약 리포트

### 최근 수정사항

- `roterfaden.kr` 첫 화면에서 바로 로그인할 수 있도록 로그인 폼을 배치했습니다.
- 로그인 후 프로필 설정 화면을 거치지 않고 권한별 작업공간으로 바로 이동하도록 변경했습니다.
- 관리자 메뉴명을 `Settings`, 화면 제목을 `Set Management`로 변경했습니다.
- 운영 옵션 관리 저장 후 성공/실패 메시지를 정확히 표시하도록 수정했습니다.
- 글구조와 루브릭 평가구조에 관리자가 직접 프롬프트 설명을 입력할 수 있게 했습니다.
- 루브릭 가중치 메뉴를 추가했습니다.
- 관리자 Reports에서 학생별, 세션별, 튜터별, 생성된 문장별 필터를 추가했습니다.
- 관리자 Sessions에서 튜터별 필터를 추가했습니다.
- Texts에서 구조 분석 JSON을 사람이 읽는 HTML 형태로 표시합니다.
- 관리자 Classes는 생성/저장 없이 참조 중심으로 정리하고, 삭제 시 `확인` 입력 안전장치를 추가했습니다.
- 글구조/루브릭 평가구조의 프롬프트 설명 입력창은 펼침/닫힘 방식으로 정리했습니다.
- 루브릭 가중치 숫자 입력칸에서 0을 Backspace/Delete로 지울 수 있도록 수정했습니다.
- 튜터 Sessions에서 튜터 필터와 상세 튜터 표시를 제거했습니다.
- 튜터 Sessions 상세의 `초안`, `공개`, `종료` 상태 버튼은 선택된 상태만 빨간색으로 표시되도록 정리했습니다.
- 튜터 Sessions 삭제 시 확인 창을 거치도록 안전장치를 추가했습니다.
- 튜터 제출물 리뷰에서 `수정 저장`을 `최종 저장`으로, `최종 공개`를 `피드백 보내기`로 변경했습니다.
- 최종 저장 전에는 `피드백 보내기`를 비활성화하고, 최종 저장 후에는 `최종 저장`을 잠근 뒤 `피드백 보내기`를 활성화하도록 정리했습니다.
- 튜터 Reports의 학생 리포트 화면에서 튜터 항목을 제거했습니다.
- 학생 화면도 공통 왼쪽 메뉴와 로그아웃 박스 구조로 통일했습니다.
- 보호자 화면도 공통 왼쪽 메뉴와 로그아웃 박스 구조로 통일했습니다.
- 학생 스케치 이미지 제출 시 `submission_images`와 Storage RLS가 충돌하지 않도록 `014_submission_image_rls_fix.sql`을 추가했습니다.
- 제출 화면, 포트폴리오, 피드백 상세의 깨진 한국어 문구를 정리했습니다.

## 운영 체크

- Supabase 프로젝트에는 `supabase/migrations`의 SQL 파일을 `014_submission_image_rls_fix.sql`까지 순서대로 적용합니다.
- Vercel 환경 변수에는 Supabase URL, anon key, service role key, OpenAI API key가 입력되어 있어야 합니다.
- 배포 후 `/`, `/login`, `/manual`, `/admin/users`, `/tutor/workbench`, `/tutor/sessions`, `/tutor/reports`, `/student/dashboard`, `/parent/dashboard`를 확인합니다.
