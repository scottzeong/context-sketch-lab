# Roter Faden 사용자 매뉴얼 및 릴리즈 노트

## 제품 개요

Roter Faden은 튜터가 수업용 글을 작성하고, 학생이 맥락 스케치를 제출하며, 튜터가 피드백과 리포트를 관리하는 학습 운영 도구입니다.

v1에서 AI의 역할은 다음으로 제한합니다.

- 튜터가 정한 주제와 조건에 맞는 글 생성
- 튜터가 직접 입력한 글의 구조 분석
- 저장된 글의 구조 분석
- 튜터 관찰 내용을 바탕으로 한 피드백 평가 초안 생성

AI는 학생 그림을 최종 판정하지 않습니다. 학생 스케치 해석과 최종 피드백 결정은 튜터가 담당합니다.

## 역할별 사용 흐름

### 관리자

1. `/admin/users`에서 튜터, 학생, 보호자 계정을 생성합니다.
2. 사용자 role, 표시 이름, 학습 연령대, 계정 상태를 관리합니다.
3. 보호자와 학생을 연결하고 승인 상태로 변경합니다.
4. 비밀번호 재설정 이메일을 발송하거나 계정을 비활성화합니다.
5. 글 작성 화면의 연령, 난이도, 분량, 글구조 드롭다운 메뉴를 관리합니다.
6. Supabase SQL migration 적용 여부와 Vercel 환경 변수를 점검합니다.

### 튜터

1. `/tutor/workbench`의 Step 1 `글 작성`에서 `글 생성` 또는 `글 입력`을 선택합니다.
2. AI가 생성한 글 또는 튜터가 붙여 넣은 글을 구조 분석합니다.
3. 결과가 적절하면 Text 저장소에 저장합니다.
4. `/tutor/texts`에서 저장된 글을 검색하고 확인합니다.
5. `/tutor/groups`에서 학습 그룹을 만들고 학생을 배정합니다.
6. `/tutor/sessions/new`에서 글, 그룹, 활동지 유형을 선택해 세션을 생성합니다.
7. `/tutor/submissions`에서 학생 제출물을 검토하고 튜터 관찰을 입력합니다.
8. AI 피드백 평가 초안을 생성한 뒤 튜터가 수정하고 공개합니다.
9. `/tutor/reports`에서 학생별 리포트 초안을 편집, 저장, 인쇄/PDF 출력합니다.

### 학생

1. `/student/dashboard`에서 공개된 세션을 확인합니다.
2. 세션 글을 읽고 종이에 맥락 스케치를 작성합니다.
3. 제출 화면에서 스케치 이미지를 업로드하고 자기 설명을 입력합니다.
4. 튜터가 피드백을 공개하면 피드백 상세 화면에서 확인합니다.
5. `/student/portfolio`에서 공개 피드백과 성장 기록을 누적 확인합니다.

### 보호자

1. 관리자에게 학생과의 연결 승인을 요청합니다.
2. `/parent/dashboard`에서 연결된 학생의 공개 피드백을 확인합니다.
3. 학생별 필터로 최근 피드백과 누적 기록을 봅니다.
4. 보호자 요약 리포트를 인쇄하거나 PDF로 저장합니다.

## 릴리즈 노트 v1

### 핵심 기능

- 브랜드 프론트 페이지 `/`
- HTML 매뉴얼 페이지 `/manual`
- Supabase Auth 기반 role 라우팅
- 관리자 계정 생성 및 권한 관리
- 관리자 글 작성 드롭다운 설정
- 튜터 AI Workbench
- AI 글 생성 및 튜터 직접 글 입력
- Text 저장소
- 학습 그룹 관리
- 세션 생성, 공개, 종료, 복제
- 학생 제출 UX
- 튜터 리뷰 워크스페이스
- 학생 피드백 상세 화면
- 학생 포트폴리오
- 튜터 리포트 편집/저장/인쇄/PDF
- 보호자 대시보드 및 요약 리포트

### 최근 수정사항

- 로그인 전 `/`을 로그인 화면이 아닌 브랜드 프론트 화면으로 변경했습니다.
- 프론트 문구를 `AI Agent의 시대 : 정보의 미로 속에서 스스로 길을 찾는 여정`으로 변경했습니다.
- 로그인, 매뉴얼, 프로필 설정 화면의 제목 크기를 조정했습니다.
- 프로필 설정과 관리자 사용자 관리 화면에서 읽기 수준 입력 항목을 제거했습니다.
- 관리자 로그인 후 기본 이동 위치를 `/admin/users`로 변경했습니다.
- 왼쪽 메뉴와 브랜드 영역 크기를 조정했습니다.
- 운영 화면 하단의 `v1 planning build` 표시와 Tutor Dashboard의 `Next Build` 영역을 제거했습니다.
- hover 시 버튼 글자가 사라지지 않도록 링크 색상을 보정했습니다.

## 운영 체크

- 새 Supabase 프로젝트에는 `supabase/migrations/011_config_options.sql`까지 순서대로 적용합니다.
- Vercel 환경 변수에 Supabase URL, anon key, service role key, OpenAI API key가 입력되어야 합니다.
- 배포 후 `/`, `/login`, `/manual`, `/admin/users`, `/tutor/workbench`, `/student/dashboard`, `/parent/dashboard`를 확인합니다.
