# 튜터 매뉴얼

## 전체 흐름

```text
글 작성
→ Text 저장
→ 구조 분석
→ 세션 생성/공개
→ 학생 제출 확인
→ AI 피드백 초안 생성
→ 튜터 수정
→ 최종 저장
→ 피드백 보내기
→ 리포트 작성
```

## 글 작성 Workbench

1. `/tutor/workbench`로 이동합니다.
2. `TEXT ARCHITECTURE`에서 `AI Generation` 또는 `Text Writing`을 선택합니다.
3. `AI Generation`은 주제, 연령, 난이도, 분량, 글 구조, 학습 목표를 바탕으로 AI 초안을 만듭니다.
4. `Text Writing`은 튜터가 직접 작성했거나 준비한 글을 텍스트 박스에 붙여 넣습니다.
5. Generated Text 영역에 글이 올라오면 `구조분석` 버튼이 활성화됩니다.
6. 구조 분석이 완료되면 Structure Analysis 영역에서 `저장`합니다.

피드백 초안 생성은 Workbench가 아니라 `/tutor/submissions`의 리뷰 화면에서 처리합니다.

## Text 저장소

- URL: `/tutor/texts`
- 제목, 본문, 목표, 구조, 난이도로 검색합니다.
- 상태, 구조, 난이도 필터로 자료를 정리합니다.
- 저장된 글과 구조 분석 결과를 사람이 읽기 쉬운 형태로 확인합니다.
- 삭제 버튼을 누르면 한 번 더 확인한 뒤 삭제합니다.

## Class 관리

- URL: `/tutor/groups`
- 수업 Class를 생성합니다.
- 학생을 Class에 배정합니다.
- 세션은 Class 단위 또는 전체 학생 대상으로 배포할 수 있습니다.

## 세션 생성

1. `/tutor/sessions/new`로 이동합니다.
2. 사용할 글을 검색하고 선택합니다.
3. 세션 제목, Class, 학습 목표, 활동지 템플릿을 확인합니다.
4. 상태를 `초안` 또는 `공개`로 선택합니다.
5. 저장합니다.

## 세션 관리

- URL: `/tutor/sessions`
- 제목, 글, Class, 목표로 검색합니다.
- Class와 상태 필터를 사용합니다.
- 세션 목록과 상세 설명 영역에는 튜터 필터/튜터 정보가 표시되지 않습니다.
- 상세 설명 영역의 `초안`, `공개`, `종료`는 현재 선택된 상태만 빨간색으로 표시되고 나머지는 흰색으로 표시됩니다.
- 상태 버튼을 누르면 해당 상태로 변경됩니다.
- `복제`로 반복 수업용 세션을 빠르게 만들 수 있습니다.
- 삭제 버튼을 누르면 확인 창이 뜨며, 확인 후에만 세션이 삭제됩니다.

## 제출물 리뷰

1. `/tutor/submissions`에서 제출물을 선택합니다.
2. 학생이 업로드한 스케치 이미지를 확인합니다.
3. 학생 설명, 핵심 연결, 어려웠던 부분을 읽습니다.
4. 튜터 관찰, 핵심 연결, 강점, 오해/보완점, 다음 과제를 입력합니다.
5. `AI 피드백 초안`을 눌러 초안을 생성합니다.
6. 학생 피드백, 튜터 메모, 보호자 요약을 튜터가 직접 수정합니다.
7. `최종 저장`으로 최종 문구를 저장합니다.
8. 최종 저장 전에는 `피드백 보내기`가 비활성화됩니다.
9. `최종 저장`을 누르면 최종 저장 버튼은 빨간색 상태로 잠기고, `피드백 보내기`가 활성화됩니다.
10. `피드백 보내기`를 누르면 튜터가 수정한 최종본만 학생과 보호자에게 공개됩니다.

주의:

- AI 초안은 최종 평가가 아닙니다.
- 학생 그림 해석과 피드백 공개 책임은 튜터에게 있습니다.
- 공개 전 학생에게 보일 문구와 보호자 요약을 반드시 확인합니다.

## 리포트

- URL: `/tutor/reports`
- 학생, 세션, 생성된 문장, Class, 기간으로 필터링합니다.
- 학생 리포트 상세 화면에는 튜터 정보가 표시되지 않습니다.
- 공개 피드백 기반 리포트 초안을 편집합니다.
- `초안 저장`으로 Supabase에 저장합니다.
- `인쇄/PDF`로 출력합니다.
