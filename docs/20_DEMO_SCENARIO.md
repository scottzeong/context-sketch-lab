# Roter Faden 데모 시나리오

## 목적

이 문서는 1차 개발 완료 QA와 외부 시연을 위한 표준 데모 흐름입니다.

## 준비 계정

```text
admin@example.com
tutor@example.com
student@example.com
parent@example.com
```

실제 데모에서는 Supabase Auth에 생성된 실제 이메일을 사용합니다.

## 데모 1. 관리자 운영

1. 관리자로 로그인합니다.
2. `/admin/users`로 이동합니다.
3. 튜터, 학생, 보호자 계정이 있는지 확인합니다.
4. 학생 계정에 연령대와 읽기 수준이 있는지 확인합니다.
5. 보호자-학생 연결을 `승인` 상태로 저장합니다.
6. 필요하면 비밀번호 재설정 이메일 버튼을 보여줍니다.

성공 기준:

- 사용자 목록이 보인다.
- role과 계정 상태를 수정할 수 있다.
- 보호자-학생 연결이 저장된다.

## 데모 2. 튜터 글 생성과 저장

1. 튜터로 로그인합니다.
2. `/tutor/workbench`로 이동합니다.
3. 주제, 분량, 글 구조를 입력해 글을 생성합니다.
4. 구조 분석을 실행합니다.
5. 결과가 수업에 적합하면 Text 저장소에 저장합니다.
6. `/tutor/texts`에서 저장된 글을 검색합니다.

성공 기준:

- AI 생성 결과가 사람이 읽는 형태로 표시된다.
- 원본 JSON은 기본 화면에서 과하게 노출되지 않는다.
- 저장된 글을 검색/필터할 수 있다.

## 데모 3. 그룹과 세션 생성

1. `/tutor/groups`에서 학습 그룹을 확인합니다.
2. 학생이 그룹에 포함되어 있는지 확인합니다.
3. `/tutor/sessions/new`로 이동합니다.
4. 저장된 글을 검색하고 선택합니다.
5. 그룹과 활동지 템플릿을 선택합니다.
6. 세션을 `공개` 상태로 저장합니다.
7. `/tutor/sessions`에서 세션을 확인합니다.
8. 세션 복제 버튼으로 반복 수업 초안을 만듭니다.

성공 기준:

- 세션이 공개 상태로 저장된다.
- 그룹/상태/검색 필터가 동작한다.
- 복제된 세션은 초안 상태다.

## 데모 4. 학생 제출

1. 학생으로 로그인합니다.
2. `/student/dashboard`에서 공개된 세션을 선택합니다.
3. 글을 읽습니다.
4. 제출 화면으로 이동합니다.
5. 스케치 이미지와 자기 설명을 제출합니다.

성공 기준:

- 학생은 자기 그룹에 공개된 세션을 볼 수 있다.
- 이미지 업로드가 성공한다.
- 제출 후 튜터 제출 목록에 나타난다.

## 데모 5. 튜터 리뷰와 피드백 공개

1. 튜터로 로그인합니다.
2. `/tutor/submissions`에서 제출물을 선택합니다.
3. 학생 이미지와 설명을 확인합니다.
4. 튜터 관찰 내용을 입력합니다.
5. AI 피드백/평가 초안을 생성합니다.
6. 학생용 피드백과 보호자 요약을 수정합니다.
7. 피드백을 공개합니다.

성공 기준:

- AI 초안은 튜터가 수정 가능한 형태다.
- 피드백 공개 후 상태가 바뀐다.
- 학생과 보호자가 공개 피드백을 볼 수 있다.

## 데모 6. 학생/보호자 확인

1. 학생으로 로그인합니다.
2. `/student/portfolio`에서 공개 피드백을 확인합니다.
3. 보호자로 로그인합니다.
4. `/parent/dashboard`에서 연결된 학생의 피드백을 확인합니다.
5. 보호자 요약 리포트 인쇄 버튼을 확인합니다.

성공 기준:

- 학생은 자기 피드백만 본다.
- 보호자는 연결 승인된 학생 피드백만 본다.
- 부모용 요약이 자연스럽게 표시된다.

## 데모 7. 리포트 저장과 출력

1. 튜터로 로그인합니다.
2. `/tutor/reports`로 이동합니다.
3. 학생, 그룹, 기간 필터를 사용합니다.
4. 리포트 초안을 수정합니다.
5. `초안 저장`을 누릅니다.
6. `인쇄/PDF`를 누르고 출력 화면을 확인합니다.

성공 기준:

- 리포트 초안이 저장된다.
- 새로고침 후에도 저장된 초안이 유지된다.
- 인쇄/PDF 화면이 읽기 좋게 정리된다.

## 데모 후 점검

- Production 로그에 치명적 오류가 없는가?
- Supabase Storage에 제출 이미지가 저장되었는가?
- `report_drafts`에 리포트 초안이 저장되었는가?
- role별 접근 제한이 유지되는가?
