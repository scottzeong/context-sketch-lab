# 최종 개발 및 QA 체크리스트

## 1차 개발 완료 기준

Production에서 아래 흐름을 1회 이상 성공하면 1차 개발 완료로 판단합니다.

```text
관리자 계정 생성
→ 튜터/학생/보호자 계정 생성
→ 보호자-학생 연결 승인
→ 관리자 운영 옵션과 루브릭 설정
→ 튜터 글 작성/분석/저장
→ 튜터 그룹 생성 및 학생 배정
→ 튜터 세션 생성/공개
→ 학생 제출
→ 튜터 리뷰/피드백 공개
→ 학생 포트폴리오 확인
→ 보호자 대시보드 확인
→ 튜터 리포트 저장/인쇄/PDF 확인
```

## 기능 QA

- 관리자는 `/admin/users`에서 계정을 생성할 수 있다.
- 새 계정은 `/onboarding`에서 프로필과 비밀번호를 설정할 수 있다.
- 비활성 계정은 서비스 화면 접근이 차단된다.
- 보호자-학생 연결을 승인/대기/해제할 수 있다.
- 관리자는 연령, 난이도, 분량, 글구조, 루브릭 평가구조, 루브릭 가중치를 관리할 수 있다.
- 글구조와 루브릭 평가구조의 프롬프트 설명이 저장된다.
- 루브릭 가중치가 레벨별로 저장된다.
- 튜터는 AI 글 생성과 직접 글 입력을 모두 사용할 수 있다.
- AI 글 생성 분량은 요청 글자수의 약 ±10% 이내를 목표로 한다.
- 글 저장 후 구조 분석 버튼이 활성화된다.
- Text 저장소에서 구조 분석이 HTML 형태로 보인다.
- 그룹 생성 후 학생을 배정할 수 있다.
- 관리자 그룹 화면은 생성/저장 없이 참조 중심이며 삭제 시 `확인` 입력이 필요하다.
- 세션 생성 화면에서 글 검색과 그룹 선택이 동작한다.
- 세션 목록에서 검색/그룹/상태/튜터 필터가 동작한다.
- 학생은 공개된 세션만 볼 수 있다.
- 학생 제출 이미지가 Supabase Storage에 저장된다.
- 튜터가 제출물 이미지를 보고 리뷰를 작성할 수 있다.
- AI 피드백 초안을 생성하고 튜터가 수정할 수 있다.
- 최종 공개 피드백만 학생과 보호자에게 보인다.
- 리포트는 학생별, 세션별, 튜터별, 생성된 문장별로 필터링된다.
- 리포트 초안을 편집/저장/복사/인쇄할 수 있다.

## 권한 QA

- 튜터는 `/admin/users`에 접근할 수 없다.
- 학생은 `/tutor/*`, `/admin/*`, `/parent/*`에 접근할 수 없다.
- 보호자는 `/student/*`, `/tutor/*`, `/admin/*`에 접근할 수 없다.
- 보호자는 승인된 학생의 공개 피드백만 볼 수 있다.
- 학생은 다른 학생의 제출물과 피드백을 볼 수 없다.
- 비활성 계정은 RLS helper 기준에서도 organization 데이터 접근이 제한된다.
- Storage 이미지는 제출 학생, 같은 조직 staff, 승인된 보호자만 접근할 수 있다.

## 모바일 QA

- 로그인/온보딩 화면에서 입력 필드가 잘리지 않는다.
- 학생 대시보드와 제출 화면이 모바일에서 자연스럽게 정렬된다.
- 튜터 세션/텍스트/리포트 필터가 모바일에서 겹치지 않는다.
- 보호자 대시보드 카드와 인쇄 버튼이 모바일에서 사용할 수 있다.
- 주요 버튼 텍스트가 버튼 밖으로 나가지 않는다.
